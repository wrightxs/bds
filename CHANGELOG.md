# CHANGELOG

## 2026-06-24

### 新增
- **Tushare 数据源**: 完整实现 `TushareFetcher`，通过 Tushare Pro API 获取全市场日线数据，股票名称改从数据库获取避免 `stock_basic` 限频
- **仪表盘数据源切换**: 标题旁新增下拉框（AKShare/Tushare），运行时即时切换，默认 Tushare
- **强制刷新**: Tushare 模式下日期选择器右侧新增红色按钮，`force=true` 清除重抓
- **均线突破页签**: 替换原右侧交易逻辑，基于 MA10/MA20/MA30 均线策略筛选
  - 10天：底部区域 + 首次突破10日线
  - 20天：底部区域 + 站上10日线
  - 30天：底部区域 + 金叉（同时站上三条均线）
- **缠论页签**: 缠论1买（底部背驰反转）和2买（回踩不破前低）检测，成交额>1亿过滤，默认展示2买
- **仪表盘新指标**: 跌停数量、破板数、破板率替换原概览卡片
- **日期导航**: 所有页面日期选择器两侧新增← →按钮，快速切换交易日
- **环比变化**: 当日卡片显示较前日增幅，前N日卡片显示各自环比
- **一键部署脚本**: `deploy.sh` 导出数据库→打包→上传→启动
- **镜像加速**: Dockerfile（APT清华+pypi清华+npm npmmirror），`.npmrc`
- **CHANGELOG.md**: 本文件

### 修改
- **均线突破**: 从低点反弹策略改为均线策略
- **数据源默认值**: `config.py` 默认改为 `tushare`
- **仪表盘概览卡片**: 市场总成交额/涨停数/成交额榜首 → 跌停数/破板数/破板率
- **日期刷新**: `fetchDates()` 每天仅请求一次（内存缓存），每次 `loadData` 前先刷新
- `.env` 从 git 移除（含 token），加入 `.gitignore`

### 风险项
- Tushare 免费版 `stock_basic` 接口限频 1次/小时，已改为从数据库获取股票名称
- 数据库仅 6 个交易日数据，均线计算使用近似值（`min(period, len(prices))`），数据积累后自动提升精度
- 缠论检测在数据量少时低点识别不够准确，随数据增加改善

### 遗留问题
- 历史数据（6/16-6/18）可能重复，需用 force=true 重抓验证
- 均线突破 LOOKBACK 当前 30 天，数据超 30 天后建议改为 60
- 无自动化测试
